# 1. Git 의미
**Git이란?** 소스 코드와 문서의 변경 이력을 추적하고, 여러 개발자가 동시에 작업할 수 있게 돕는 분산 버전 관리 시스템

# 2. Git 흐름 이해
**원격 동기화**     
- 원격 변경을 로컬로 가져옴   
- ex) git pull / git fetch + git merge
    
**작업 브랜치**     
- 기능/수정 작업을 분리   
- ex) git checkout -b feature
    
**스테이징**
- 커밋할 변경을 준비
- ex) git add / git add -A
    
**커밋**
- 스냅샷을 로컬 저장소에 기록
- ex) git commit -m
    
**푸시**
- 로컬 커밋을 원격에 올림
- ex) git push / git push -u
    
**병합/리베이스**
- 다른 브랜치와 이력 통합
- ex) git merge / git rebase
    
**PR/리뷰**
- 변경을 검토 및 합치기 요청
- ex) Github PR

# 3. Git flow
**의미**    
Vincent Driessen의 브랜칭 모델을 위한 고수준 저장소 작업을 제공하는 git의 확장 / 기능 개발과 릴리스 준비를 브랜치 단위로 체계화해 협업을 돕는다
    
## 3-1. 주요 명령어
**초기화** :  flow init (브랜치 명명 규칙을 묻고 기본값 사용 권장)
    
**기능시작** : git flow feature start MYFEATURE (develop 기반으로 새 브랜치 생성 및 이동)
    
**기능 종료** : git flow feature finish MYFEATURE (MYFEATURE를 develop에 병합하고 삭제)
    
**릴리스 시작** : git flow release start RELEASE (develop에서 release 브랜치 생성)
    
**릴리스 종료** : git flow release finish RELEASE (release를 main에 병합하고 태그 생성)
    
**특징**
- git flow는 병합 기반 
- 릴리스 준비를 별도 브랜치로 분리해, 다음 개발과 동시에 현재 릴리스를 다듬을 수 있게 설계
- 팀 규모가 커지면 브랜치 세분화가 유용
- PR을 통해 코드 리뷰와 CI 검증을 병해하는 방식 권장

# 4. GitHub 의미
Git 저장소를 온라인에서 호스팅하고, 여러 사람이 함께 코드를 공유 및 협업 할 수 있게 해줌

# 5. Git VS GitHub
| **구분** | **Git** | **GitHub** |
| --- | --- | --- |
| **정의** | 버전 관리 시스템 | Git 저장소 호스팅 서비스 |
| **주 사용처** | 로컬에서 변경 이력 관리 | 원격 저장소 공유, 협업, 백업 |
| **협업 방식** | 로켈에서만 가능 | 원격 저장소를 통해 공동 작업 가능 |
| **대표 기능** | 커밋, 브랜치, 병합 등 | 이슈, PR, Actions, Pages 등 협업 도구 |
